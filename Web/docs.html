Getting Started

TG Drive maps your Telegram cloud account as a native virtual disk on Windows. Follow the instructions to install the client and start syncing files immediately.

System Requirements

To run TG Drive, you need: Windows 10 or Windows 11 (64-bit), a secure internet connection, and an active Telegram account.

Installation

1. Download the latest installer executable from our official website or GitHub releases.

2. Run the TelegramDriveSetup.exe installer. No administrator privileges are required.

3. The setup extracts all application binaries under your local app data directory and creates desktop/start-menu shortcuts.

Configuration & Mapping

Setting up secure authorization is a one-time process. All credentials remain completely local and encrypted.

Getting Telegram API Credentials

Before logging in, you must obtain an API ID and API Hash from Telegram:
  1. Go to my.telegram.org and sign in with your phone number.
  2. Select API development tools.
  3. Create a new application (you can use any name/shortname).
  4. Copy the generated App api_id and App api_hash.

Client Login

Launch TG Drive, enter your API ID, API Hash, and phone number (in international format, e.g. +123456789). Click 'Connect'. Enter the verification code (OTP) sent to your Telegram application.

Mount Options & Drive Letters

By default, TG Drive mounts as the virtual letter T: (Telegram Cloud). You can change this letter in settings if it conflicts with a physical disk. The client also lets you configure a local downloads directory to cache files temporarily for faster retrieval, and select automatic background backup folders to sync files from your local PC to Telegram.

Key Features & Architecture

SQLite Database & Storage Logs

TG Drive uses an embedded, encrypted SQLite database locally on your system to maintain index structures, settings, mappings, and activity logs. You can review synchronization history and debug logs directly within the app.

Process Cleanup Guard

To avoid mounting errors or database locks, the client runs a single-instance startup check. If any previous zombie process of TG Drive is still running in the background, it will automatically shut down and clean up active handles before booting the new window.

Cache-Busting & Retries

Built over the high-performance WTelegramClient client library, the application integrates customized network resilience mechanisms. If a connection dropout occurs, the sync engine automatically performs cache-busting retries with exponential backoffs to prevent file transfer corruption.

Troubleshooting & FAQs

Security Notice

Never share your API ID, API Hash, or verification code (OTP) with anyone. TG Drive will never request this information online or send it to external servers; it is stored solely on your local computer.

Error: Multiple Instances Detected

If the app reports that another session is active, close TG Drive completely. Check Windows Task Manager to kill any lingering TelegramDrive.exe tasks manually if the automated cleanup fails.

Telegram Connection/Auth Errors

Ensure your API ID and Hash are correct. A common error is a mismatch in phone format. Ensure you include the full country prefix (e.g., +1, +44, +213) and do not insert any spaces. Also verify that you have not exceeded Telegram's API call rate limits.

SQLite Database Lock Issues

If files fail to sync and log errors mention database locking, restart the application. This happens if the SQLite file is accessed by multiple worker threads simultaneously. The built-in engine automatically handles lock retries, but a restart will release all database connections.
